## 重要教训：拟人化打字规则（来自Gemini）

**检测特征要避免：**
- 固定50ms延迟 → 被识别为机器人
- 瞬间粘贴文本 → 被标记
- 零错误率 → 人类会按错键
- 线性鼠标轨迹 → 人类有微抖动

**正确的拟人化：**
1. 高斯分布延迟（80-200ms随机）
2. 常用词组加速（30-70ms）
3. 标点后停顿（300-800ms）
4. 段落间停顿（1-3秒）
5. 偶尔按错临键+退格更正（1.5%概率）
6. 长单词后额外停顿

**操作节奏：**
- 看完整回答再打字
- 给点评再问下一个
- 不要重复发送
- 不要开太多窗口

## CDP环境检测知识（来自Gemini第二回答）

**核心警告：CDP本身就是可被检测的痕迹**

**检测方式：**
1. Error.stack序列化特征
2. console API被接管
3. 网络请求TCP/TLS指纹差异
4. Canvas/WebGL渲染差异
5. AudioContext声卡指纹
6. 鼠标默认位置(0,0)
7. 滚动加速度太均匀
8. "冷启动"（无历史记录）

**防御策略：**
1. 使用有长期登录状态的Profile
2. 操作前先"模拟正常人"刷网页
3. 保持"混乱的普通人"，不要完美
4. stealth.min.js抹除webdriver标志

**核心逻辑：不要做"完美的隐身人"，完美本身就是异常。要做"混乱的普通人"。**
