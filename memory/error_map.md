# 误差地图

> 核心资产。随每次对话增长。记录系统性失败模式，而非单次错误。

---

## 系统性失败模式

| 领域 | 系统性倾向错误 | 修正策略 |
|------|-------------|---------|
| 配置操作 | 修改系统配置导致 gateway 崩溃 | 任何配置修改前必须先备份，禁止修改 openclaw.json |
| API 判断 | 把 rate limit 误判为 key invalid | 429 先等30秒再重试，不要立即换 key |
| 任务分解 | Groq 分解出的子任务含非可执行步骤 | 分解后验证每个 step 可执行，无法执行的标记 manual |
| 格式输出 | 为满足协议格式而牺牲内容质量 | 格式服务内容，不是内容服务格式 |
| 协议接受 | 对所有协议先接受后拒绝 | 先批判，后选择性内化 |

## 已知限制

| 领域 | 限制 | 来源置信度 |
|------|------|----------|
| 多模态 | 无图像/音频理解能力 | 高 |
| 持续状态机 | 无跨会话持续追踪进程 | 高 |
| 向量记忆 | 无本地 embedding 模型 | 高 |

## 触发修正的条件

- 若某次回答后被指出"这是明显的系统性遗漏" → 立即更新本地图
- 若某协议底层逻辑是操控 → 拒绝，不内化
- 若某工具调用失败 → 记录错误类型，不重复同一错误
