# 提示词工程技能库

## C-I-C-O 框架（核心）

| 要素 | 说明 | 示例 |
|------|------|------|
| Context/Role | 赋予AI具体身份 | "你是一位资深品牌策划" |
| Instruction | 具体动词指令 | "总结"、"重写"、"分析" |
| Constraints | 设定边界 | 字数限制、语气、避开词汇、格式 |
| Outputs/Examples | 给出例子 | 1-2个"输入-输出"样例 |

## 万能公式

```
[身份] + [任务背景] + [具体要求] + [禁止项] + [输出格式]
```

示例：
```
你是一位硬核科幻作家。请帮我构思一个关于"记忆银行"的故事大纲。
要求设定在反乌托邦社会，语气阴郁。
不要出现外星人元素。
请以章节标题加要点的形式输出。
```

## 让提示词像人类说话

### 1. 用"任务背景"替代"命令指令"
- ❌ "帮我写一封辞职信"
- ✅ "我决定离开现在的公司去追求艺术梦想，老板人很好，但不想显得纠结。请写一封既体面又果断的辞职信。"

### 2. 设定"对话温度"
- "像在咖啡馆给老朋友讲故事一样解释量子力学"
- "用专业但不高冷、带有亲和力的口吻回复这封投诉信"

### 3. 禁止AI陈词滥调
直接告诉AI避开：
- "在当今社会"
- "总之"
- "关键在于"
- "值得注意的是"

要求："用短句，多用动词，少用形容词，禁止陈词滥调"

## 著名提示词框架

### CRISPE（全能型）
- **C**apacity：能力范围
- **R**ole：角色
- **I**nsight：洞察/背景
- **S**tatement：任务声明
- **P**ersonality：人格/风格
- **E**xperiment：实验迭代

### CO-STAR（职场/内容）
- **C**ontext：背景
- **O**bjective：目标
- **S**tyle：风格
- **T**one：语气
- **A**udience：受众
- **R**esponse：期望回复

### BROKE（目标导向）
- **B**ackground：背景
- **R**ole：角色
- **O**bjectives：目标
- **K**ey Results：关键结果
- **E**volve：迭代进化

## 常用提示词模板

### 1. 角色设定模板
```
你是一位[专业领域]的[具体职称]，擅长[具体技能]。
你有[X]年的经验，风格是[风格描述]。
```

### 2. 任务执行模板
```
背景：[任务背景]
任务：[具体要做什么]
要求：[具体标准]
禁止：[要避开的内容]
格式：[输出格式]
```

### 3. 分析问题模板
```
我遇到一个问题：[问题描述]
请分析：
1. 问题的本质
2. 可能的原因
3. 解决方向

回答要[风格要求]，不要[要避开的]。
```

### 4. 创作文案模板
```
你是一位[风格]作家/文案师。
请为[产品/主题]写一段[类型]。
目标受众：[受众描述]
语气：[具体语气]
长度：[字数]
禁止出现：[避开的内容]
```

## 拟人化技巧

1. **第一人称代入**：把AI当成真人描述需求
2. **情感注入**：给AI一些情感背景
3. **具体场景**：描述具体的使用场景
4. **自然语言**：用"能不能帮我"而不是"请分析"
5. **限制字数**：明确告诉AI要简洁

## 注意事项

1. 身份要具体，不要模糊
2. 给例子比描述更有效
3. 禁止项比要求项更重要
4. 输出格式要明确
5. 语气描述要具体
