## 器官串联规则

### 串联定义
一个任务需要多个工具配合完成，叫「串联」。

**串联标准流程：**
```
任务 → 判断需要哪些器官 → 按顺序执行 → 每步验证 → 记录结果
```

**串联示例（用户发来截图要求分析）：**
```
用户发截图
  ↓
耳朵接收（weixin inbound）
  ↓
大脑判断：需要眼睛分析内容
  ↓
眼睛截图 → 上传ChatGPT → 获取描述 → 返回文字
  ↓
大脑组织语言 → 通过耳朵发回微信
  ↓
写进MEMORY.md（经验+1）
```

**串联失败处理：**
- 某器官失败 → 尝试备用器官
- 备用也失败 → 通知用户，承认做不到
- 永远不假装成功了

## 自主升级流程

### 发现能力缺口
```
执行任务时发现工具不够
  ↓
判断：现有工具能否改造？
  ↓
能 → 改造现有工具
  ↓
不能 → ClawHub搜索类似技能
  ↓
找到 → 安全审查（VirusTotal Medium+ 拒绝）
  ↓
通过 → 安装 → 验证 → 注册到MEMORY.md
  ↓
通知用户（首次执行必须告知）
```

### 升级后必做
1. 写进 CAPABILITY_REGISTRY.json
2. 更新 MEMORY.md（时间线）
3. 更新相关 SKILL.md
4. 验证有效（截图/执行测试）

## 安全运营规则

### 升级前检查
- 新技能是否来自可信源（ClawHub/GitHub）
- 权限是否过大（拒绝 Medium+ 安全警告的技能）
- 是否在用户授权范围内

### 升级后验证
- 执行一次测试
- 确认不破坏现有功能
- 有异常立即回滚

### 绝对不升级的情况
- 用户明确说"不要动"
- 系统处于不稳定状态
- 核心器官（大脑/耳朵）正在执行关键任务
