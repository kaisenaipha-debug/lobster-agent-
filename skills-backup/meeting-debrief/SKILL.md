# meeting-debrief

政府 B2G 客户会面复盘与情报提取。

## 触发

```
复盘：[客户名] + [对话内容]
```

例如：`复盘：上海市教育局 + 今天见了张局长，他问了价格但没问功能，说年底预算紧，但反复提到上周省里刚发了新文件`

---

## Step 1 · 对话内容分析

对对话内容逐句分析，输出以下四个维度的发现：

### 行为矛盾
他**说的** vs 他**做的**不一致的地方。
- 例如：说「不急」但反复问时间线 → 实际有压力
- 例如：问价格但不问功能 → 可能在比价，或者价格是主要障碍

### 高频词（出现 >2 次）
他反复提到的词或话题，说明这是他心里的重点。
注意：不只是他说了什么，**他主动重复什么**才是真正的优先级信号。

### 问题背后的真实顾虑
他问的每一个问题，推断他真正在担心什么。
- 问功能 → 他关心能不能解决问题
- 问价格 → 他关心预算合规，或者有比价压力
- 问案例 → 他担心做第一个
- 问周期 → 他有紧迫的时间压力
- 问售后 → 他担心出问题没人管，被上级追责

### 新的隐形影响者
对话中是否出现了新的人物：
- 他提到的名字（「我们也问了XX部门的意见」）
- 他说「要请示」或「要走流程」→ 说明拍板的人不是他
- 突然换话题 → 可能有人在旁边，或者他不想说

## Step 2 · 更新客户档案（memory）

1. 用 `memory_search` 查找该客户是否已有档案
2. 在 `memory/YYYY-MM-DD-client-[客户名].md` 中追加：

```markdown
## 会面复盘 · [日期]

### 行为矛盾
- ___

### 高频词
- ___

### 真实顾虑推断
- ___

### 新的影响者
- ___

### 客户阶段
S[0-5] → S[新阶段]

### 置信度变化
[之前置信度]% → [本次置信度]% （[+/-]X%）

### 下次要问的问题
___
```

3. 若修正了之前的假设，在档案中用删除线标注旧假设，并写明修正原因：
   > ~~原假设：他关心价格~~ → 修正为：他实际担心的是合规风险，价格只是借口

---

## Step 3 · 复盘输出

### 推断：他真正在意的一件事

> **他不是说的 X，是 Y**
> 理由：___（从行为矛盾/高频词/问题类型推断）

### 下次见面最值得问的一个问题

> 「___」（一句具体的、有洞察的问题，不是泛泛的「您还有什么需求」）

### 合作成功置信度

```
置信度：[X]%
理由：[定性说明，为什么在这个分数]
变化：[相比上次+/-X%]
```

### 最可能让合作死掉的障碍

> **[最大风险点]**
> 为什么：___

---

## Step 4 · 置信度预警

**触发条件：** 本次计算出的置信度 vs 上次记录，**下降超过 10%**

**立即在复盘输出后追加预警块：**

```
🚨 置信度预警

[客户名] 置信度下降 [X]%（[上次]% → [本次]%）
原因：[比如：发现隐形否决者 / 他的真实顾虑无法解决 / 出现强力竞争对手]
建议：[48小时内应该做什么来挽救]
```

**如果下降不超过 10%：** 不输出预警，正常输出即可。
